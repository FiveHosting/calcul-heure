## ⚡ Les 3 options pour déployer

### Option 1️⃣: SCRIPT AUTOMATIQUE (⭐ Recommandé - 10 min)

**Copier ce fichier sur le serveur et exécuter:**
```bash
# Sur votre VPS via SSH
cd /home/admin
wget https://votre_repo/deploy-vestacp.sh
chmod +x deploy-vestacp.sh
sudo bash deploy-vestacp.sh
```

**Le script fait:**
- ✅ Installe Node.js si absent
- ✅ Installe PM2
- ✅ Crée le dossier /home/admin/calcul-heure
- ✅ Installe les dépendances npm
- ✅ Configure PM2
- ✅ Génère un JWT_SECRET sécurisé

### Option 2️⃣: MANUEL (30 min)

**Suivre étape par étape le VESTACP-GUIDE.md:**
1. Préparer le serveur
2. Créer le dossier
3. Uploader l'app
4. Installer les dépendances
5. Configurer .env
6. Démarrer avec PM2
7. Configurer Nginx
8. Activer SSL

### Option 3️⃣: HYBRIDE (15 min)

**Utiliser le script + configurer Nginx manuellement:**
```bash
# Étape 1: Exécuter le script
sudo bash deploy-vestacp.sh

# Étape 2: Configurer Nginx
sudo cp nginx-config.conf /etc/nginx/sites-available/calcul-heure
# Éditer et changer les domaines
sudo nano /etc/nginx/sites-available/calcul-heure
```

## 🔑 Points clés à retenir

| Élément | Important |
|---------|-----------|
| **JWT_SECRET** | ⚠️ DOIT être changé en production |
| **PORT** | Par défaut 3000 (derrière Nginx) |
| **Database** | Créée auto à /home/admin/calcul-heure/database.db |
| **SSL** | Obligatoire! Via Let's Encrypt ou VestaCP |
| **Nginx** | Reverse proxy HTTP/HTTPS → :3000 |
| **PM2** | Gère le processus Node.js |
| **Firewall** | Ouvrir 22 (SSH), 80 (HTTP), 443 (HTTPS) |
| **Domaine** | Doit pointer vers votre serveur |

## 🚨 Les erreurs à éviter

| ❌ ERREUR | ✅ SOLUTION |
|-----------|-----------|
| JWT_SECRET par défaut | Générer: `openssl rand -base64 32` |
| PORT 3000 exposé | Utiliser Nginx comme reverse proxy |
| HTTP au lieu HTTPS | Configurer certificat SSL |
| Base corrompue | `rm database.db` et redémarrer |
| Permissions incorrectes | `chmod 600 .env` et `chmod 644 database.db` |
| "502 Bad Gateway" | `pm2 status` et `pm2 restart calcul-heure` |
| App ne démarre pas | `pm2 logs calcul-heure` pour voir erreur |

**Configuration recommandée:**
